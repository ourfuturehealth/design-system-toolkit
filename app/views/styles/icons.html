Our rules for using icons are

  • Only use icons when there’s a real user need for them (avoid unnecessary decoration)
  • Use icons sparingly - most people won’t understand an icon vocabulary
  • If used, ensure icons are clear, simple and accompanied by relevant text
  • Don’t hide functionality under icons

User need

As a user I need to easily identify, understand and navigate content, so that I get the best outcome for my situation and don't endanger my health.

How to use icons

We inline SVG icons for performance and to have the ability to control the appearance of icons with CSS. This decision was inspired by CSS Tricks - A Pretty Good SVG Icon System and CSS Tricks - A Complete Guide to SVG Fallbacks.

Using a server side or templating language you can include the icons with include 'partials/icons/icon-search.svg'.

SVG icons have the class .nhsuk-icon which set a default size for the icon and each icon has a specific class such as .nhsuk-icon__search which can modify specific icons with styles such as fill. This means you can change the colour of the SVG for states such as hover, focus and active.

Each SVG icon has a PNG fallback which means we can support browsers such as Internet Explorer 8. The fallback images have a class such as .nhsuk-icon__search--fallback which can be used to modify sizing and spacing of the PNG fallbacks.

SVG icons and logos must be accessible, read more about accessibile SVGs on CSS Tricks website.
