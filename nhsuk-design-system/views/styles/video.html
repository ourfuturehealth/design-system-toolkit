Rules

  • Only use video media where it adds to user understanding
  • Don't be overly reliant on video to convey information - from an accessibility perspective, providing content as a video can limit access to the information contained in this format for anyone who cannot see or hear the content. In addition, videos break. In testing we’ve seen many a user run across a video that won’t load, doesn’t appear, can’t be played, or freezes. (https://www.nngroup.com/articles/video-usability/)
  • Video sizing - to ensure video flows with content; size 2/3 on larger screens (desktop / tablet) and full width on small screens (mobile)
  • Title and description - do not overlay video, must sit within content.
  • Clearly display the length of the video

User needs

  • As a user I WANT visual context (supporting written information), SO THAT I can help identify my health thing (condition / symptom / medicine).

Assumptions to test

  • Users want to know how long the video is upfront
  • Users don't understand review dates
  • Users don't want overlaid text
